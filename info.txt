Проверка текущего пользователя
whoami        # покажет текущее имя пользователя
id            # покажет uid/gid и группы

Рекомендуемый способ — использовать sudo

(без ввода пароля root, безопасно и широко используется)

sudo -i       # откроет интерактивную root-сессию (как вход root)
sudo -s       # похожо, но не загружает полностью окружение root
sudo <команда>  # выполнить одну команду от имени root, например: sudo apt update


sudo -i — лучше, если нужен полноценный root-интерфейс. sudo <команда> — предпочтительно для единичных задач.

На Ubuntu и многих дистрибутивах обычный пользователь с правом sudo — стандарт.

Если нужен root через su

(требуется пароль root; часто используется в RHEL/CentOS)

su -          # переключиться на root, если известен его пароль
su - username # переключиться на другого пользователя

Если root заблокирован / нет пароля (Ubuntu)

Чтобы установить/разблокировать пароль root (не рекомендуется без необходимости):

sudo passwd root    # задаёт пароль для root


После этого можно su - с введённым паролем.

Быстрая проверка, что ты root
whoami    # должно вывести "root"
id -u     # должно вывести "0"

Как выйти из root
exit
# или Ctrl+D

Примеры для конкретных случаев

В Ubuntu / Cloud Shell (обычно у тебя есть sudo):

sudo -i


В CentOS/RHEL (обычно используют root-пароль или sudo):

su -          # если знаешь пароль root
# или
sudo -i

Безопасность — важное замечание

По возможности исполняй отдельные команды через sudo <команда>, а не работай постоянно под root.

Не оставляй root-пару открытой без необходимости.
